  • In the cleaned dataset (df1_clean_new) which is stored in the twitter_archive_master.csv file, the number of columns have increased. The additional data gathered querying the Twitter API has been added to this file. The additional data that was gathered using Tweepy were the 'retweet_count' and 'favorite_count'. Using this data, it was helpful to understand how many tweets of the @weratedogs account were retweeted and how many people liked (favorite_count) these tweets.

  • In the same dataset (df1_clean_new), the ratings show the variations from 1 to 20, whereas before cleaning, the rating_numertor showed some wrongful and misguiding data. Same is the case with rating_denominator, that after cleaning all the denominators are 10, but before cleaning the dataset, the rating_denominators were some random numbers for most of the rows in the dataset.

  • There were lot of errors seen in the name column before cleaning the dataset. After cleaning the dataset, the names such as "a, an, the, such, quite" were replaced to "None" so that data consistency and completeness issues will be satisfied. Data consistency and completeness are a couple of the important issues to be consisdered while tidying the data.
